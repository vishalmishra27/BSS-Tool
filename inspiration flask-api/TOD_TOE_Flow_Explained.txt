================================================================================
  TOD → TOE END-TO-END FLOW  —  Evidence Filtering Behavior
================================================================================

OVERVIEW
--------
The system runs two sequential audit phases:
  1. TOD (Test of Design)    — evaluates whether each control is well-designed
  2. TOE (Test of Effectiveness) — evaluates whether each control actually operates

The key optimization: TOE only extracts and evaluates evidence for controls that
PASSED in TOD, since failed designs don't need operating effectiveness testing.

There are TWO code paths that run TOE:
  A) Flask API endpoints (/api/tod-test, /api/toe-test)  — api.py
  B) SOX Agent tools (test_of_design, test_of_effectiveness) — agent/tools/

Both paths now filter to TOD-passed controls only.


================================================================================
  PATH A: FLASK API FLOW
================================================================================

STEP 1: POST /api/tod-test
─────────────────────────────────────────────────────────────────────────────────
  File: api.py lines 611–772

  a) Loads TOD evidence for all controls
  b) Runs TOD evaluation → returns list of TODControlResult (PASS/FAIL/NO EVIDENCE)
  c) Caches raw results for the TOE endpoint:
       latest_results['_tod_raw_results'] = results
       latest_results['_tod_raw_schemas'] = schemas


STEP 2: POST /api/toe-test
─────────────────────────────────────────────────────────────────────────────────
  File: api.py lines 782–970

  a) Reads cached TOD results
  b) Builds include_control_ids = {only TOD-PASS control IDs}
  c) Loads evidence with filter → skips DI extraction for non-PASS controls
  d) Runs TOE with tod_results + pre_schemas → second layer of filtering
  e) Full RCM universe maintained via placeholder results

  Fallback: If no cached TOD results → extracts everything (old behavior)

================================================================================
  THE ROOT CAUSE OF THE SLOWNESS
================================================================================

The user reported: "TOE is taking a lot of time despite only 2 controls passing TOD"

Root cause: Phase 3 (evidence loading + LLM evaluation) was already filtered.
But Phase 2 (evidence validation) was NOT filtered. This meant:

  - If RCM had 10 controls and only 2 passed TOD:
    - Phase 2 would extract ALL 10 controls' evidence files via Document Intelligence
    - Phase 2 would run embeddings for ALL 10 controls' documents
    - Only Phase 3 would finally skip the 8 failed controls

  The DI extraction in Phase 2 was the bottleneck — it processes every file
  (PDF, DOCX, images, etc.) using Azure Document Intelligence, which takes
  several seconds per file. For 8 unnecessary controls with multiple samples
  each, this could add minutes of wasted time.

After the fix:
  - Phase 1: Doc list generated for 2 controls only
  - Phase 2: DI extraction + embeddings for 2 controls only
  - Phase 3: Evidence loading + LLM evaluation for 2 controls only

  3. RCM controls with no TOE evidence folder (lines 3781-3809):
     operating_effectiveness = "No Evidence"
     deficiency_type = "Unable to Assess"
     overall_remarks = "Evidence folder not found. Manual review required."

  Final summary confirms full universe:
    "Controls: {count} (RCM universe) | Evaluated: {n} | Samples: {n}"

TOE Behavior (after fix):

    C-004  → Placeholder: "No Evidence — TOE not performed"
    C-005  → Placeholder: "TOD Failed — TOE not required"

  Final results: 5 controls (full RCM universe)
