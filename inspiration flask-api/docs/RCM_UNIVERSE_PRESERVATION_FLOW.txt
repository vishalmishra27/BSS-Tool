================================================================================
  RCM UNIVERSE PRESERVATION — END-TO-END FLOW DOCUMENTATION
================================================================================

  This document describes the complete flow from RCM loading to TOE completion,
  with specific focus on the changes made to preserve the full RCM universe
  in all outputs (no controls are silently dropped).

  Key Principle: Every control in the normalized RCM MUST appear in both
  TOD and TOE final outputs, regardless of missing evidence or prior failures.

================================================================================
  OVERVIEW OF CHANGES
================================================================================

  Files Modified:
  ---------------
  1. engines/TOD_Engine.py
     - test_all_tod(): Appends placeholder results for missing-evidence controls
     - export_tod_workpaper(): Handles "NO EVIDENCE" result type
     - test_all_toe(): Accepts tod_results param, skips TOD-failed controls

  2. engines/TOE_Engine.py
     - test_all_toe(): Same changes as TOD_Engine

  3. agent/tools/test_of_design.py
     - Updated summary display to include "NO EVIDENCE" count

  4. agent/tools/test_of_effectiveness.py
     - Passes state.tod_results to engine
     - Updated summary display for new result categories

  5. engines/run_tod_toe.py
     - Passes tod_results between TOD and TOE runs

  STEP 1: RCM LOADING (agent/tools/rcm_loader.py)
================================================================================

  User uploads RCM file (Excel/CSV)
           │
           ▼
  ┌─────────────────────────────────────────────────────────────────────┐
  │  RCMLoaderTool.execute()                                            │
  │                                                                     │
  │  1. Read original Excel → original_df (all columns, original names) │

================================================================================
  KEY BEHAVIORAL GUARANTEES
================================================================================

  1. FULL RCM UNIVERSE PRESERVED
     - Every control in state.rcm_df appears in TOD output
     - Every control in state.rcm_df appears in TOE output
     - No controls are silently dropped from any output

  2. TOD-FAILED CONTROLS ARE SKIPPED IN TOE
     - Even if TOE evidence exists for a TOD-failed control, it is NOT evaluated
     - Saves API calls (no LLM evaluation for controls that need redesign)
     - Placeholder result clearly explains: "TOD Failed — TOE not required"

  3. NO-EVIDENCE CONTROLS ARE CLEARLY MARKED
     - Controls without evidence folders get placeholder results
     - Clearly marked as "Unable to Assess" with explanatory remarks
     - Auditor knows exactly which controls need manual review

  4. BACKWARD COMPATIBLE
     - All existing callers (Flask API, legacy agent, tests) work unchanged
     - The tod_results parameter defaults to None
     - When None, TOE behaves exactly as before (no TOD-based skipping)
