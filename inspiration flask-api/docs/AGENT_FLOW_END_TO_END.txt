1. ARCHITECTURE OVERVIEW
================================================================================

LangGraph StateGraph with 3 nodes:

    ┌──────────┐     has tool calls?     ┌───────────┐
    │  agent   │ ──────── YES ─────────► │   tools   │
    │ (LLM)   │                          │ (execute) │
    └────┬─────┘                          └─────┬─────┘
         │  no tool calls                       │
         ▼                              ┌───────────────┐
       END                              │ extract_facts  │
                                        └───────┬───────┘
                                                └──► back to agent

LLM: Azure OpenAI GPT-5.2
Max iterations per message: ~25
Context window: last 40 messages + semantic facts + plan scratchpad


================================================================================
2. AGENT REASONING CYCLE
================================================================================

  User message arrives
       │
       ▼
  RuntimeStateManager.get_or_create(chat_id)
  → Returns or creates AgentState (restores cache from Cosmos DB)
       │
       ▼
  LangGraph graph.astream_events() — SSE stream to frontend
       │
       ▼
  AGENT NODE (call_model):
    a. Build dynamic system prompt from AgentState
    b. Retrieve relevant playbooks via embedding similarity
    c. Inject semantic facts + plan scratchpad
    d. Window messages to last 40 (safe boundary)
    e. Call LLM with all 26 tools bound
    f. TOD/TOE mismatch guard
    g. Append deterministic suggestions (replace LLM-generated)
       │
       ▼
  ROUTE: tool_calls? → TOOLS NODE → EXTRACT_FACTS → back to AGENT
         no tool_calls? → END (stream done)

SSE events: token, tool_start, tool_end, done, error

3. STATE MANAGEMENT
================================================================================

AgentState (per session, in-memory):

  RCM:           rcm_df, original_rcm_df, output_dir

                 sampling_results, comparison_results
  TOD/TOE:       pending_tod_schemas, pending_toe_schemas, tod_schemas,
                 tod_evidence_folder, pending_tod/toe_evidence_folder
  Documents:     document_list_rows, evidence_validated
  Scoping:       scoping_engine, scoping_phase, scoping_trial_balance_path,
                 scoping_benchmark, scoping_materiality_pct, scoping_sop_paths,
                 scoping_quantitative_results, scoping_awaiting_input,
                 scoping_progress
  Progress:      tool_progress
  Session:       tool_call_count, plan_scratchpad, tool_results_cache, artifacts

RuntimeStateManager: per-thread, LRU eviction, Cosmos DB restore on creation


================================================================================
4. MEMORY SYSTEM (3-tier)
================================================================================

  Tier 1 — Episodic: full conversation (windowed to last 40 messages)
  Tier 2 — Semantic: auto-extracted facts (survives windowing, max 15)
           Examples: "RCM loaded: 42 rows", "TOD: 8 PASS, 2 FAIL"
  Tier 3 — Plan: free-form markdown scratchpad (LLM-maintained)


================================================================================
5. PATH A — RCM-FIRST WORKFLOW
================================================================================

  1. Load RCM (load_rcm) → Inspect (inspect_dataframe)
  2. AI Gap Analysis (run_ai_suggestions) → Merge (merge_suggestions)
  3. Deduplication (run_deduplication) → Remove (remove_duplicates)
  4. Control Assessment (run_control_assessment) — optional
  5. Test of Design (TOD) — 3-phase, see Section 7
  6. Sampling (run_sampling_engine) — REQUIRED between TOD and TOE
  7. Test of Effectiveness (TOE) — 3-phase, see Section 8
  8. Save & export (save_excel)

Natural sequence: Load → [Gap] → [Dedup] → TOD → Sampling → TOE → Save


================================================================================
6. PATH B — SCOPING-FIRST WORKFLOW
================================================================================

Phase-by-phase with user review at each step.
At every phase: show results → upload prompt → user can download/modify/re-upload.

  Step 1:  Ask for trial balance file path
  Step 2:  Ingest (parse, normalise, categorise, benchmarks)
           → Show accounts + upload prompt → STOP
  Step 2b: User uploads modified Excel (iterate) or continues
           → Ask for benchmark + materiality_pct
  Step 3:  Quantitative analysis (materiality threshold, classify)
           → Show results + upload prompt → STOP
  Step 3b: Upload or continue
  Step 4:  Qualitative analysis (risk factors, scoring)
           → Show results + upload prompt → STOP
  Step 4b: Upload or continue
  Step 5:  Final scoping (7-rule matrix + LLM Rule 8 overrides)
           → Show in-scope list + upload prompt → STOP
  Step 5b: Upload or continue → ask for SOP/policy file paths
  Step 6:  Downstream (process mapping, SOP extraction, RCM generation)
           → Show RCM workbook
  Step 7:  Auto-load generated RCM → suggest TOD as next step
           → Continue with Path A from step 5

           │
           ▼
  ┌─ 4. PHASE 1 — Document List Generation ────────────────────────────┐
  │                                                                      │
  │ User approves → agent calls run_test_of_design() [NO evidence]     │
  │                                                                      │
  │ Inside:                                                              │
  │   a. Check state.pending_tod_schemas exists                         │
  │   b. Call generate_document_list_excel():                            │
  │      - For each control: LLM generates required documents list     │
  │      - Parallel (5 workers)                                         │
  │      - Writes styled Excel:                                         │
  │        [Control ID | Description | Attributes | Docs | Doc Descs]  │
  │   c. Cache rows in state.document_list_rows                         │
  │   d. Return: phase="document_list", awaiting_evidence_folder=True  │
  │                                                                      │
  │ Agent shows: document list Excel path                                │
  │ Agent asks: "Please provide the TOD evidence folder path."          │
  └──────────────────────────────────────────────────────────────────────┘
           │
           ▼
  ┌─ 5. User Provides Evidence Folder ──────────────────────────────────┐
  └──────────────────────────────────────────────────────────────────────┘
           │
           ▼
  ┌─ 6. PHASE 2 — Evidence Validation ─────────────────────────────────┐
  │                                                                      │
  │ Agent calls: run_test_of_design(evidence_folder="/path/...")         │
  │                                                                      │
  │ Inside:                                                              │
  │   a. Scan evidence files per control subfolder                      │
  │   b. Create embeddings (ONE TIME) for evidence files + content      │
  │      previews using text-embedding-ada-002                          │
  │   c. Create embeddings for each Required Document from Phase 1     │
  │   d. Cosine similarity: each document vs each evidence file         │
  │   e. Build match report:                                             │
  │                                                                      │
  │      C-P2P-001:                                                      │

  │   [Re-upload Folder] — fix evidence, provide new path, re-validate │

  ┌─ 7. PHASE 3 — Run Actual TOD Test ─────────────────────────────────┐
  │                                                                      │

  │                                                                      │
  │ Inside:                                                              │

  │   c. Build pre-approved schemas from state.pending_tod_schemas      │

  │   e. Cache: state.tod_results, state.tod_schemas                    │
  │   f. Clear: state.pending_tod_schemas = None                        │
  │   g. Export: 4_TOD_Results.xlsx → upload to Blob Storage            │
  │                                                                      │
  │ Agent shows: "TOD: X PASS, Y FAIL out of Z controls"               │
  │ UI buttons: [Run Sampling] [Analyse TOD Results] [Save TOD Results] │
  └──────────────────────────────────────────────────────────────────────┘

10. EVIDENCE VALIDATION
================================================================================

Module: agent/tools/evidence_validator.py
Function: validate_evidence_against_documents()
NOT a standalone tool — called internally by TOD/TOE Phase 2.

How it works:
  1. Scan evidence folder — files per control subfolder
  2. Create embeddings (ONE TIME) for:
     - Evidence files: "filename: content_preview"
     - Required Documents: "name: description"
     Using Azure OpenAI text-embedding-ada-002 (batch, 16 per call)
  3. Cosine similarity: each required document vs each evidence file
     Threshold: 0.45 (semantic — "PO_12345.pdf" matches "Purchase Order")
  4. Build per-control match report

Output:
  C-P2P-001:  3/5 matched, 2 missing
    ✅ Purchase Order              ← "PO_12345.pdf"    (0.87)
    ✅ Goods Receipt Note          ← "GRN_receipt.pdf"  (0.79)
    ✅ Vendor Invoice              ← "Invoice_V001.pdf" (0.82)
    ❌ Three-Way Match Report      ← NOT FOUND
    ❌ Exception Approval Email    ← NOT FOUND

User options when documents missing:
  [Re-upload Folder] — provide new/fixed evidence folder path
  [Skip & Continue]  — proceed with skip_validation=true

12. SUGGESTIONS ENGINE
================================================================================

Module: agent/core/suggestions.py
Deterministic — replaces LLM-generated suggestions.

State priority order:
  1. Scoping intermediate → upload prompt
  2. Pending schemas → [Modify] [Upload Excel] [Approve & Run]
  3. Scoping complete → [Analyse] [Review RCM] [Run TOD]
  4. After TOE → [Compare Results] [Save Workpaper]
  5. After Sampling → [Run TOE] [Save]
  6. After TOD → [Run Sampling] + [Remove Failed] + [Analyse] + [Save]
  7. RCM loaded → context-specific (based on last_tool)

13. REFLECTOR
================================================================================

Module: agent/core/reflector.py
Injects _agent_notes into tool results. Does NOT control flow.

  - Phases 1 & 2 (document_list, evidence_validation): skipped
  - Repeated tool calls (3+): loop warning
  - High tool count (>15): wrap-up suggestion
  - TOD 100% pass: anomaly flag
  - TOD >50% fail: systemic warning
  - TOE zero deviations: anomaly flag
  - TOE not effective: PCAOB finding
  - Dedup zero pairs in large RCM: threshold note
  - Sampling unmatched frequencies: warning
  - Scoping re-call: blocked

14. TOOL ADAPTER & SLIM RESULTS
================================================================================

Module: server/tool_adapter.py
Wraps agent/tools as LangChain StructuredTools for LangGraph.

Phase 1 & 2 results (document_list, evidence_validation): passed through as-is.
Phase 3 (full test) results: slimmed for token efficiency.

Slim tools:
  load_rcm                  → strips rcm_rows
  run_test_of_design        → strips per-control results (keeps pass/fail + doc list)
  run_test_of_effectiveness → strips per-control details (keeps effectiveness + doc list)
  run_sampling_engine       → strips per-control results (keeps totals)
  run_sox_scoping_engine    → strips account lists (keeps counts + sample)

Guards: tool call limit, read_file→load_rcm redirect, preconditions

17. SERVER ARCHITECTURE
================================================================================

FastAPI + Uvicorn

Startup:
  1. Cosmos DB (8 containers)
  2. Azure Blob Storage + cache cleanup daemon
  3. LangGraph SQLite checkpointer
  4. Compile LangGraph agent
  5. RuntimeStateManager
  6. Azure OpenAI connectivity probe
  7. Idle state eviction task (5 min)

Key endpoints:
  POST /api/chats/{id}/messages     — send message (SSE stream)
  GET  /api/chats/{id}/messages     — get message history
  GET  /api/chats/{id}/scoping-progress   — live scoping progress
  GET  /api/chats/{id}/tool-progress      — live TOD/TOE progress
  GET  /api/chats/{id}/session-results    — cached results (page refresh)
  GET  /api/chats/{id}/workflow-state     — workflow state
  POST /api/chats/{id}/tod-approval       — edited TOD schemas from UI
  POST /api/chats/{id}/toe-approval       — edited TOE schemas from UI
  POST /api/upload                        — file upload to Blob
  POST /api/upload-folder                 — folder upload (evidence)
  GET  /api/download?path=...             — download artifact
  GET  /api/health                        — health check

Security: CORS, internal API token guard, health endpoints open.
